### 需要掌握的主要测试类型及侧重点

| 测试类型       | 概念（是什么）            | 侧重点（关注什么）          |
| ---------- | ------------------ | ------------------ |
| **配置测试**   | 使用各种硬件平台测试软件运行的过程  | 硬件兼容性、不同配置下的表现     |
| **兼容性测试**  | 检查软件之间是否能正确交互和共享信息 | 平台版本、标准规范、数据共享     |
| **外国语言测试** | 测试软件在不同语言/地区的适应性   | 翻译、本地化、配置兼容性       |
| **易用性测试**  | 测试软件是否易于理解、操作和使用   | UI规范性、辅助选项（残障支持）   |
| **安全性测试**  | 测试软件抵御攻击的能力        | 缓冲区溢出、威胁建模、漏洞      |
| **网站测试**   | 对Web应用进行多层次测试      | 黑盒/灰盒/白盒、配置兼容性、易用性 |
| **功能测试**   | 详尽测试每个功能是否正确       | 能力、健壮性、功能组合（CRUD）  |
| **性能测试**   | 测试系统响应时间、吞吐量等指标    | 速度、资源占用、稳定性        |

## 一、配置测试（Configuration Testing）

表格

| 维度      | 说明                                                                                              |
| :------ | :---------------------------------------------------------------------------------------------- |
| **概念**  | 通过多种**软件和硬件组合**来测试应用程序，以找出系统在无缺陷或错误的情况下可以工作的**最佳配置**，同时验证添加或修改硬件资源（如内存、磁盘、CPU、网卡）对系统性能的影响。      |
| **侧重点** | ① 确定硬件和软件的**最小配置**和**最佳配置**；② 评估添加/修改资源对系统性能的影响；③ 发现软硬件组合中的兼容性问题；④ 通常由程序员在开发/调试阶段执行，属于动态白盒测试范畴。 |

> **与兼容性测试的区别**：配置测试关注"在特定配置下软件能否正常工作"，而兼容性测试关注"软件能否与各种外部对象（浏览器、硬件、其他软件）协同工作"。配置测试通常不在干净的系统环境下进行，兼容测试多在干净环境下进行。

## 二、兼容性测试（Compatibility Testing）

表格

|维度|说明|
|:--|:--|
|**概念**|检查软件在不同**硬件平台、软件平台、操作系统、浏览器、网络环境**等条件下是否可以正常运行，即软件的可移植性和互操作性。<br><br>[](http://mp.weixin.qq.com/s?__biz=MzkwNjQ0Mjc4Mg==&mid=2247487703&idx=2&sn=f76840e5c9575769c4dcb152e003dc24)|
|**侧重点**|① **平台兼容**：不同 OS（Windows/Linux/Mac/Android/iOS/鸿蒙）；② **浏览器兼容**：Chrome、Safari、Edge、Firefox 等；③ **硬件兼容**：CPU、内存、显卡、打印机等；④ **网络兼容**：WiFi/2G/3G/4G/5G/无网；⑤ **前后向兼容**：软件版本升级/降级兼容性；⑥ **数据兼容**：不同系统间的数据格式互通。<br><br>[](https://blog.csdn.net/qq_36616591/article/details/133934968)|

## 三、外国语言测试（国际化/本地化测试）

表格

| 维度      | 说明                                                                                                                                             |     |
| :------ | :--------------------------------------------------------------------------------------------------------------------------------------------- | --- |
| **概念**  | 包括**国际化测试（I18N）**和**本地化测试（L10N）**：国际化是设计软件使其能适应不同语言和地区而无需修改；本地化是将国际化软件适配到特定地区或语言。                                                              |     |
| **侧重点** | **国际化**：① 处理外文文本和数据（排序、导入、大小写处理）；② 正确处理货币、日期、时间格式；③ 从源代码中分离字符串；④ UI 预留足够空间显示外语。**本地化**：① 软件翻译准确性；② 图标和图形是否符合当地文化；③ 帮助文档和手册翻译；④ 地区特定的法规和文化习惯适配。 |     |

## 四、易用性测试（Usability Testing）

表格

|维度|说明|
|:--|:--|
|**概念**|验证软件是否**用户友好**，是否易于最终用户理解、学习和操作，属于人体工程学范畴，关注用户与系统的交互体验。|
|**侧重点**|① **易理解性**：控件名称是否易懂、术语是否专业恰当；② **易学习性**：是否有帮助文档、操作提示；③ **易操作性**：快捷键支持、Tab 键切换、回车确认、取消/回退功能；④ **吸引性**：界面美观、布局合理；⑤ **容错性**：错误提示是否友好、是否支持撤销；⑥ **响应性**：操作响应时间是否合理。<br><br>[](https://blog.csdn.net/qq_36616591/article/details/133934968)|

## 五、安全性测试（Security Testing）

表格

|维度|说明|
|:--|:--|
|**概念**|评估软件保护信息和数据的能力，验证系统是否能防止未授权访问、数据泄露、恶意攻击等安全威胁。|
|**侧重点**|① **保密性**：数据加密传输与存储；② **完整性**：防止数据被篡改；③ **认证与授权**：权限控制、身份验证机制；④ **不可抵赖性**：操作日志与审计追踪；⑤ **抗攻击性**：SQL 注入、XSS、CSRF 等漏洞检测；⑥ **可核查性**：安全策略是否可验证。<br><br>[](https://blog.csdn.net/qq_36616591/article/details/133934968)|

## 六、网站测试（Website Testing）

表格

|维度|说明|
|:--|:--|
|**概念**|针对 Web 应用的专项测试，涵盖功能、性能、安全、兼容性、可用性等多个维度，确保网站在不同浏览器、设备和网络环境下正常运行。|
|**侧重点**|① **功能测试**：链接有效性、表单提交、购物车、支付流程；② **UI/布局测试**：不同分辨率下的页面渲染、响应式设计；③ **兼容性测试**：跨浏览器、跨设备；④ **性能测试**：页面加载速度、并发访问能力；⑤ **安全测试**：HTTPS、会话管理、敏感数据保护；⑥ **SEO 测试**：搜索引擎优化友好性；⑦ **可访问性测试**：残障用户辅助功能支持。|

## 七、性能测试（Performance Testing）

表格

|维度|说明|
|:--|:--|
|**概念**|确定或验证被测系统或应用程序的**速度、可扩展性和稳定性**特征，关注响应时间、吞吐量、资源利用率等性能指标。|
|**侧重点**|① **响应时间**：用户请求到系统响应的耗时；② **吞吐量**：单位时间内处理的请求数；③ **资源利用率**：CPU、内存、磁盘 I/O、网络带宽占用；④ **并发能力**：同时在线用户数；⑤ **稳定性**：长时间运行是否出现内存泄漏或性能衰减。<br><br>[](https://blog.csdn.net/qq_36616591/article/details/133934968)|

**性能测试的子类型**：

|子类型|侧重点|
|:--|:--|
|**负载测试**|验证系统在预期负载下的行为，确定最大可持续负载|
|**压力测试**|将系统置于超出极限的负载下，直到崩溃，观察恢复能力|
|**容量测试**|验证系统处理大量数据的能力|
|**稳定性/ soak 测试**|高负载下长时间运行，检测内存泄漏等问题|
|**并发测试**|多用户同时访问，检测死锁和资源竞争|
