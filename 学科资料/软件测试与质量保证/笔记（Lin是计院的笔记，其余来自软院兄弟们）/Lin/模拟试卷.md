### 四、分析计算题（共40分）

#### 第1题（10分） 单元测试与Google Test

某函数 `bool isLeapYear(int year)` 判断是否为闰年。规则：能被4整除但不能被100整除的是闰年；能被400整除的也是闰年。
（1）使用等价类划分法列出闰年和非闰年的有效等价类。（4分）
（2）使用Google Test框架，写出测试闰年（如2020）和非闰年（如2021）的两个TEST宏示例代码（只写TEST部分，包含断言）。（6分）

#### 第2题（12分） 白盒测试与圈复杂度

给出以下C代码片段：

```c
void process(int a, int b, int c) {
    if (a > 0 && b < 10) {
        if (c == 5) {
            printf("Case1");
        } else {
            printf("Case2");
        }
    } else {
        printf("Case3");
        if (a == 0 || c > 0) {
            printf("Case4");
        }
    }
}
```

（1）画出该代码的控制流图（用节点表示基本块，边表示跳转）。（6分）
（2）计算圈复杂度（写出两种方法）。（4分）
（3）要达到分支覆盖，至少需要几个测试用例？请给出一个满足分支覆盖的测试用例集。（2分）

#### 第3题（10分） 性能测试概念分析

某电商系统要求：支持最大并发用户数1000，95%的请求响应时间小于2秒，系统吞吐量不低于5000笔/分钟。现执行测试，结果如下：
- 当并发用户从100逐渐增加到1000时，响应时间从0.5秒线性增加到2.5秒。
- 当并发达到1100时，系统出现部分请求超时（>5秒），但未崩溃。
- 当并发达到1200时，系统返回大量500错误，需重启服务。
请回答：
（1）该系统的性能瓶颈可能在什么负载附近出现？依据是什么？（3分）
（2）上述测试中，哪个阶段属于**负载测试**，哪个属于**压力测试**？（4分）
（3）根据结果，系统是否满足给定的性能需求？为什么？（3分）

#### 第4题（8分） 自动化测试ROI计算

某团队有200个回归测试用例，每个手工执行需要5分钟。计划开发自动化脚本，每个用例脚本平均开发时间30分钟，维护成本为每次回归每用例1分钟（即总维护成本=200×1×回归次数），执行时间忽略。
（1）写出手工总成本和自动化总成本关于回归次数n的公式。（2分）
（2）求盈亏平衡点n（手工成本=自动化成本）。（3分）
（3）若项目计划发布后只做2次回归，是否实施自动化？为什么？（3分）

## 📌 参考答案与评分要点（补充卷）

**选择题**：1.C  2.D  3.B  4.B  5.D  6.A  7.B  8.A（N=35, n=14, V=35*log2(14)）  9.B  10.C
**判断题**：1.×  2.√  3.×  4.×（开源免费）  5.×（通常需要用户点击或访问恶意链接） 6.×  7.√  8.√  9.×  10.√
**简答题**（要点）：
1. 语句覆盖：易达到但忽略分支和条件组合；分支覆盖：比语句覆盖强，但忽略复合条件中的子条件。
2. 步骤：①确定性能目标 ②设计测试方案（场景、数据） ③录制/编写脚本 ④配置线程组（并发数、过渡期） ⑤添加监听器 ⑥执行并分析结果。
3. MC/DC要求每个条件独立影响判定结果，比条件组合覆盖更严格，能发现ORF、VNF等错误。
4. 蠕虫自我复制传播（如Samy蠕虫、Santy蠕虫）；木马伪装正常程序，需用户触发（如WebShell）。
**分析计算题**（要点）：
第1题：等价类：闰年（2000、2020），非闰年（1900、2021）；TEST代码使用`EXPECT_TRUE(isLeapYear(2020))`等。
第2题：控制流图节点数约6-7，边数约8-9，圈复杂度约3；分支覆盖需覆盖所有判定结果的组合，最少2个用例（如(a>0,b<10,c=5)和(a<=0)）。
第3题：瓶颈在800-1000并发附近（响应时间增速变化）；负载测试：100→1000；压力测试：1000→1200；不满足需求（1000并发时响应时间2.5秒>2秒）。
第4题：手工成本=200×5×n=1000n；自动化成本=200×30 + 200×1×n=6000+200n；盈亏平衡点1000n=6000+200n→800n=6000→n=7.5≈8次；2次回归不应自动化（手工2000分钟，自动化6400分钟）。
