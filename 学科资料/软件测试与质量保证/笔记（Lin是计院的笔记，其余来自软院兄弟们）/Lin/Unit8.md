## ✅ **第八章：软件质量度量**

### 一、质量和度量的区别和联系

#### 1. 质量的定义（PPT中多种定义，需理解共性）

- **ISO 8402**：产品/服务的整体功能和特点，满足明确或隐含需要的能力
- **IEEE 610.12-1990**：系统/组件/过程满足客户或用户需要/期望的程度
- **GB/T 16260-1996**：软件产品满足明确或隐含需求的能力有关的特性总和
- **核心要点**：
  - 需求是度量质量的基础
  - 既要满足明确定义的需求，也要满足隐含需求
  - 质量是多种特性的复杂组合

#### 2. 度量的定义

- **度量（Metric）**：任何测量的单位（如cm、升、秒），描述实体的属性
- **软件度量**：对软件开发项目、过程及产品进行数据定义、收集、分析的持续性定量化过程
- **目的**：理解、预测、评估、控制和改善

#### 3. 质量与度量的区别和联系（重点）

| 对比项 | 质量 | 度量 |
|--------|------|------|
| **本质** | 特性、能力、满足需求的程度 | 测量、数值、量化指标 |
| **表现形式** | 定性描述（如“好用”“可靠”） | 定量数值（如复杂度10、覆盖率85%） |
| **可操作性** | 无法直接测量 | 通过度量来间接反映质量 |
| **关系** | 质量是目标，度量是手段。**没有度量，质量管理只能是幻想** |
> **核心联系**：质量需要用度量来评估。通过选取合适的度量元（如圈复杂度、缺陷密度、覆盖率等），可以量化软件质量，从而进行质量控制、预测和改进。

---


### 二、软件质量模型

软件质量由质量特性反映，质量模型是分层模型（特性 → 子特性 → 度量）。需要掌握以下三个经典模型：

#### 1. McCall质量模型（1977）

- **面向三个维度**：产品运行、产品修订、产品变迁
- **11个质量特性**（了解大致分类）：
  - 运行：正确性、可靠性、效率、完整性、可用性
  - 修订：可维护性、灵活性、可测试性
  - 变迁：可移植性、可复用性、互操作性

#### 2. Boehm质量模型（1978）

- **分层结构**，兼顾不同用户类型（最终用户、移植用户、维护人员）
- 包含硬件特性，比McCall更全面

#### 3. ISO 9126质量模型（1991，重点）

- **6个质量特性**（需记住）：
  1. **功能性**（Functionality）
  2. **可靠性**（Reliability）
  3. **可维护性**（Maintainability）
  4. **效率**（Efficiency）
  5. **可使用性**（Usability）
  6. **可移植性**（Portability）

- 每个特性下分为若干**子特性**（共21个，考试可能考1-2个例子，如：功能性的子特性包括适合性、准确性、互操作性、安全保密性、功能性的依从性）
- **特点**：从用户、开发者、管理者多角度考虑，特性间交叉性最小
> **后续标准**：ISO 9126已演变为ISO/IEC 25000系列（SQuaRE）。国内对应标准：GB/T 16260-2006（质量模型）、GB/T 18905-2002（产品评价）、GB/T 15532-2008（测试规范）、GJB 5236-2004（军用软件质量度量）。

### 三、软件度量方法

PPT中介绍了多种度量方法，按考试要求，重点掌握以下几类：

#### 1. 代码度量（静态度量）

| 度量方法                    | 核心内容                                      | 计算公式/要点                                                                                                              |
| ----------------------- | ----------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **Halstead度量**          | 基于操作符和操作数的复杂性度量                           | n1=独立操作符数，n2=独立操作数，N1=操作符总数，N2=操作数总数；<br>程序长度 N=N1+N2，程序词汇表 n=n1+n2，<br>体积 V=N*log2(n)，难度 D=(n1/2)*(N2/n2)，工作量 E=D*V |
| **功能点（Function Point）** | 基于业务功能（外部输入、输出、查询、内部文件、外部接口）测量软件规模        | 与开发语言无关，适用于估算工作量、生产力                                                                                                 |
| **代码行统计（LOC）**          | 测量代码行数（包括空白行、注释行、代码行、混合行）                 | 反映物理规模，可指出难理解模块                                                                                                      |
| **代码基本度量**              | 嵌套层次、返回点数、参数个数、局部变量数、扇入/扇出等               | 简单直观                                                                                                                 |
| **McCabe度量（圈复杂度）**      | 测量模块内独立线性路径数量                             | 三种计算方法：<br>①  v(G)=e−n+2（边-节点+2）<br>②  v(G)=判定节点数+1<br>③  v(G)=区域数<br>**意义**：预测测试最小路径数，复杂度高则易错、难维护                   |
| **McCabe扩展度量**          | 基本复杂度(ev)、模块设计复杂度(iv)、设计复杂度(S0)、集成复杂度(S1) | ev反映非结构化程度，iv反映调用耦合度，S0=∑iv，S1=S0−n+1                                                                                |

#### 2. 覆盖率度量（测试充分性指标）

| 覆盖类型 | 定义 | 特点 |
|-----------|------|-------|
| **函数入口覆盖率** | 被调用过的函数比例 | 最基本 |
| **调用对覆盖率** | 执行过的调用对比例 | 用于集成测试 |
| **语句覆盖率** | 至少执行一次的语句比例 | 最弱，忽略分支和条件组合 |
| **分支覆盖率（判定覆盖）** | 至少执行一次的条件分支比例 | 包容语句覆盖，但忽略复杂条件 |
| **MC/DC（修正条件/判定覆盖）** | 每个条件独立影响判定结果 | 航空软件标准（DO-178B），能发现ORF、VNF、ENF类错误 |

#### 3. 缺陷度量

- **缺陷密度** = 已知缺陷数量 / 产品规模（常用每千行代码缺陷数或每个功能点缺陷数）
- **缺陷管理流程**：确保每个缺陷被解决，通过趋势曲线分析过程阶段
- **度量层次**：组织级、项目级、个体级

#### 4. 其他度量（了解）

- **CoCoMo模型**：构造性成本模型，基于代码行估算工作量（人月），分为组织型、半独立型、嵌入型

### 四、补充：软件质量度量工具（了解即可）

- **McCabe IQ**：分析圈复杂度、覆盖率、数据流、可视化
- **Logiscope**：规则检查、质量评估、测试覆盖率分析

## 📌 考试要点总结（第八章）

| 知识点         | 要求                                                                                         |
| ----------- | ------------------------------------------------------------------------------------------ |
| 质量和度量的区别与联系 | 能解释：质量是特性，度量是量化手段；没有度量就无法管理质量                                                              |
| 软件质量模型      | 掌握ISO 9126的6个质量特性（功能性、可靠性、可维护性、效率、可使用性、可移植性），了解McCall和Boehm模型的基本思想                         |

| 其他          | 了解CoCoMo模型的基本形式，了解质量标准（GB/T 16260等）的存在即可                                                   |
