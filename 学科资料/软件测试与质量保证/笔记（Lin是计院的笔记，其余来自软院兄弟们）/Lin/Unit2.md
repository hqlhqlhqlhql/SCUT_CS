## ✅ **第二章：软件测试方法**

### 一、黑盒测试（重点）

- **定义**：不考虑内部结构，只关注输入输出。
- **常用方法**：
    1. **等价类划分法**
        - 有效等价类 / 无效等价类
        - 划分原则（如取值范围、集合、布尔量、规则等）
        - 设计测试用例的步骤（编号、覆盖有效类、单独覆盖无效类）
        - 实例：三角形判断问题
    2. **边界值分析法**
        - 一般边界值（4n+1）
        - 健壮边界值（6n+1）
        - 最坏情况边界值（5^n）
        - 健壮最坏情况边界值（7^n）
        - 实例：图片上传大小限制
    3. **判定表驱动法**
        - 条件桩、动作桩、条件项、动作项
        - 建立步骤：列出条件 → 列出动作 → 填表 → 简化
        - 实例：机器维修优先级判定
    4. **因果图法**。 
        - 因果关系：恒等、非、或、与
        - 约束关系：E、I、O、R、M
        - 步骤：列出原因结果 → 画因果图 → 转判定表 → 设计用例
        - 实例：第一个字符为#或*，第二个为数字

### 二、白盒测试（重点）

- **定义**：基于内部逻辑结构，测试路径、条件、循环等。
- **控制流图**：结点表示段，边表示分支。
- **覆盖类型**：
    - 语句覆盖（最弱）
    - 分支覆盖（判定覆盖）
    - 条件覆盖
    - 条件组合覆盖
    - 基本路径覆盖（圈复杂度 C = e - n + 2）
    - 循环覆盖（简单、嵌套、顺序）
    - 数据流覆盖（了解）
    - 变元覆盖（突变测试，了解）
- **覆盖分析器**：插桩 → 执行 → 生成报告

### 三、静态测试 vs 动态测试

- **静态测试**：不运行程序，评审、代码检查、走查
    - 评审步骤：计划 → 概述 → 准备 → 检查 → 返工 → 跟踪
    - 代码检查：桌面检查、代码审查、代码走查
- **动态测试**：运行程序，输入输出验证

### 四、手工测试 vs 自动化测试

- 自动化测试适用场景：重复性强、回归频繁、ROI高
- 手工测试适用场景：UI复杂、时间紧、技术不支持

### 五、黑盒 vs 白盒对比（重点理解）

- 黑盒：适合功能、验收测试，覆盖度难度量
- 白盒：适合单元、集成测试，覆盖度高但成本大
- 平衡两者：结构测试与功能测试结合


黑盒主要了解等价类划分、边界值分析法、判定表驱动分析、因果图法
白盒主要是控制流图之后（语句覆盖、分支覆盖、条件组合覆盖、循环覆盖）

## 题目 1：等价类划分 + 边界值分析（综合题）

### 题目
某保险系统根据投保人年龄（整数）计算保费，规则如下：
- 年龄范围：**1 ~ 150**（含边界）
- **1 ~ 18** 岁：保费 **100** 元
- **19 ~ 60** 岁：保费 **200** 元
- **61 ~ 150** 岁：保费 **300** 元
- 其他（<1 或 >150，或非整数）：系统提示 **"年龄输入非法"**

请用**等价类划分法**和**边界值分析法**设计测试用例。

---

### 解题过程

#### 第一步：等价类划分

| 输入条件 | 有效等价类 | 无效等价类 |
|---------|-----------|-----------|
| 年龄范围 | (1) 1~18<br>(2) 19~60<br>(3) 61~150 | (4) <1<br>(5) >150<br>(6) 非整数（如 25.5）<br>(7) 非数字（如 "abc"）<br>(8) 空值 |

#### 第二步：边界值分析（针对有效等价类）

对每个范围的**上点、离点、内点**取值：

| 等价类 | 上点 | 离点 | 内点 |
|--------|------|------|------|
| 1~18 | 1, 18 | 0, 19 | 10 |
| 19~60 | 19, 60 | 18, 61 | 30 |
| 61~150 | 61, 150 | 60, 151 | 100 |

#### 第三步：设计测试用例

| 用例编号 | 输入数据 | 预期结果 | 覆盖方法 |
|---------|---------|---------|---------|
| 1 | 10 | 保费100 | 等价类(1) + 内点 |
| 2 | 1 | 保费100 | 边界-上点 |
| 3 | 18 | 保费100 | 边界-上点 |
| 4 | 0 | 非法 | 边界-离点 + 无效类(4) |
| 5 | 19 | 保费200 | 边界-上点 |
| 6 | 30 | 保费200 | 等价类(2) + 内点 |
| 7 | 60 | 保费200 | 边界-上点 |
| 8 | 61 | 保费300 | 边界-上点 |
| 9 | 100 | 保费300 | 等价类(3) + 内点 |
| 10 | 150 | 保费300 | 边界-上点 |
| 11 | 151 | 非法 | 边界-离点 + 无效类(5) |
| 12 | 25.5 | 非法 | 无效类(6) |
| 13 | "abc" | 非法 | 无效类(7) |
| 14 | 空值 | 非法 | 无效类(8) |

> **答题技巧**：考试中如果要求"结合两种方法"，你先把等价类表格列出来，再在边界处补充用例，最后汇总成一张用例表，这样逻辑清晰、得分完整。

## 题目 2：判定表驱动分析（多条件组合题）

### 题目
某电商系统促销规则如下：
- **条件**：是否为会员（是/否）、消费金额是否 ≥500（是/否）、是否有优惠券（是/否）
- **动作**：
  - 会员且金额≥500：打 **8** 折
  - 会员且金额<500：打 **9** 折
  - 非会员且金额≥500：打 **9.5** 折
  - 非会员且金额<500：**无折扣**
  - 有优惠券：在折扣基础上**再减 50 元**
  - 无优惠券：不额外减免

请用**判定表法**设计测试用例。

---

### 解题过程

#### 第一步：列出条件桩和动作桩

**条件桩**（3个条件，每个2种取值，共 2³ = 8 种组合）：
- C1：是否为会员（Y/N）
- C2：金额是否≥500（Y/N）
- C3：是否有优惠券（Y/N）

**动作桩**：
- A1：打8折
- A2：打9折
- A3：打9.5折
- A4：无折扣
- A5：再减50元

#### 第二步：构建判定表

| 条件/规则 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
|-----------|---|---|---|---|---|---|---|---|
| C1 会员 | Y | Y | Y | Y | N | N | N | N |
| C2 ≥500 | Y | Y | N | N | Y | Y | N | N |
| C3 优惠券 | Y | N | Y | N | Y | N | Y | N |
| **A1 8折** | ✓ | ✓ |   |   |   |   |   |   |
| **A2 9折** |   |   | ✓ | ✓ |   |   |   |   |
| **A3 9.5折** |   |   |   |   | ✓ | ✓ |   |   |
| **A4 无折扣** |   |   |   |   |   |   | ✓ | ✓ |
| **A5 减50** | ✓ |   | ✓ |   | ✓ |   | ✓ |   |

#### 第三步：合并相似规则（简化）

观察发现：A5（减50）只与 C3 有关，与其他条件无关。可以按 C3 合并，但通常考试要求**先列全表再合并**，或者题目要求"简化判定表"时才合并。

**简化后**（按折扣类型合并，保留优惠券分支）：

| 规则 | 条件组合 | 动作 |
|------|---------|------|
| R1 | 会员 + ≥500 + 有券 | 8折再减50 |
| R2 | 会员 + ≥500 + 无券 | 8折 |
| R3 | 会员 + <500 + 有券 | 9折再减50 |
| R4 | 会员 + <500 + 无券 | 9折 |
| R5 | 非会员 + ≥500 + 有券 | 9.5折再减50 |
| R6 | 非会员 + ≥500 + 无券 | 9.5折 |
| R7 | 非会员 + <500 + 有券 | 无折扣再减50 |
| R8 | 非会员 + <500 + 无券 | 无折扣 |

#### 第四步：生成测试用例

| 用例编号 | 输入（会员/金额/优惠券） | 预期折扣结果 |
|---------|----------------------|------------|
| 1 | 是 / 600 / 有 | 600×0.8−50 = 430 |
| 2 | 是 / 600 / 无 | 600×0.8 = 480 |
| 3 | 是 / 400 / 有 | 400×0.9−50 = 310 |
| 4 | 是 / 400 / 无 | 400×0.9 = 360 |
| 5 | 否 / 600 / 有 | 600×0.95−50 = 520 |
| 6 | 否 / 600 / 无 | 600×0.95 = 570 |
| 7 | 否 / 400 / 有 | 400−50 = 350 |
| 8 | 否 / 400 / 无 | 400（无折扣） |

> **答题技巧**：判定表题一定要画表格！先列全组合，再合并。如果题目说"用判定表设计最少测试用例"，合并这一步就是得分点。

## 题目 3：白盒覆盖分析（语句、分支、条件组合）

### 题目
以下是一段代码，请画出**控制流图**，并分别给出满足**语句覆盖、分支覆盖、条件组合覆盖**的最少测试用例。

```python
def calculate(a, b, c):
    if a > 0 and b > 0:
        x = a + b
    else:
        x = a - b
    
    if c > 0 or x > 5:
        y = x * c
    else:
        y = x / c
    
    return y
```

---
### 解题过程

#### 第一步：画控制流图

```
        [开始]
          |
    [1] a>0 and b>0  ←── 判断1
       /     \
     真/       \假
     /         \
  [2] x=a+b  [3] x=a-b
     \         /
      \       /
       \     /
    [4] c>0 or x>5  ←── 判断2
       /     \
     真/       \假
     /         \
  [5] y=x*c  [6] y=x/c
     \         /
      \       /
       \     /
      [7] return y
```

#### 第二步：语句覆盖（Statement Coverage）

**目标**：每条语句至少执行一次。

| 用例 | a | b | c | 执行路径 | 覆盖语句 |
|------|---|---|---|---------|---------|
| 1 | 1 | 1 | 1 | 1→2→4→5→7 | 1,2,4,5,7 |

**问题**：语句 3 和 6 没执行到！需要补充：

| 用例 | a | b | c | 执行路径 | 覆盖语句 |
|------|---|---|---|---------|---------|
| 2 | -1| 1 | -1| 1→3→4→6→7 | 1,3,4,6,7 |

**最少用例**：**2 组**（用例1 + 用例2）

#### 第三步：分支覆盖 / 判定覆盖（Branch Coverage）

**目标**：每个判断的真假分支都执行一次。

- 判断1（`a>0 and b>0`）：需要真分支和假分支
- 判断2（`c>0 or x>5`）：需要真分支和假分支

| 用例 | a | b | c | 判断1 | 判断2 | 路径 |
|------|---|---|---|-------|-------|------|
| 1 | 1 | 1 | 1 | 真 | 真(c>0) | 1→2→4→5→7 |
| 2 | -1| 1 | -1| 假(a≤0) | 假(c≤0且x≤5) | 1→3→4→6→7 |

验证判断2假分支：用例2中，a=-1, b=1 → x = -1-1 = -2；c=-1 → c>0为假，x>5为假 → 整个判断为假。**满足**。

**最少用例**：**2 组**

#### 第四步：条件组合覆盖（Multiple Condition Coverage）

**目标**：每个判断中所有条件的取值组合都出现一次。

**判断1**：`a > 0 and b > 0`
- 条件：C1(a>0), C2(b>0)
- 组合数：2² = 4 种

| 组合 | C1 | C2 | 判断1结果 |
|------|----|----|----------|
| 1 | T | T | T |
| 2 | T | F | F |
| 3 | F | T | F |
| 4 | F | F | F |

**判断2**：`c > 0 or x > 5`
- 条件：C3(c>0), C4(x>5)
- 组合数：2² = 4 种

| 组合 | C3 | C4 | 判断2结果 |
|------|----|----|----------|
| 5 | T | T | T |
| 6 | T | F | T |
| 7 | F | T | T |
| 8 | F | F | F |

**设计用例**（需要覆盖全部8种组合）：

| 用例 | a | b | c | 判断1组合 | 判断2组合 | 说明 |
|------|---|---|---|----------|----------|------|
| 1 | 1 | 1 | 1 | C1=T,C2=T(组合1) | C3=T,C4=T(x=2>5?否,但c=1>0,所以C3=T) | 组合1+组合5或6 |
| 2 | 1 | -1| 1 | C1=T,C2=F(组合2) | C3=T | 组合2 |
| 3 | -1| 1 | 1 | C1=F,C2=T(组合3) | C3=T | 组合3 |
| 4 | -1| -1| 1 | C1=F,C2=F(组合4) | C3=T | 组合4 |
| 5 | 1 | 1 | -1| 组合1已覆盖 | C3=F,C4=T(x=2>5?否) → 不行 | 需要x>5 |
| 5'| 1 | 1 | -6| 组合1 | x=2, C3=F,C4=F → 组合8 | 组合8 |
| 6 | 6 | 1 | -1| C1=T,C2=T | x=7>5, C3=F,C4=T → 组合7 | 组合7 |

整理后**最少用例**（可优化合并）：

| 用例 | a | b | c | x值 | 覆盖组合 |
|------|---|---|---|-----|---------|
| 1 | 1 | 1 | 1 | 2 | 判断1(1), 判断2(5:C3=T) |
| 2 | 1 | -1| 1 | 2 | 判断1(2), 判断2(5) |
| 3 | -1| 1 | 1 | -2 | 判断1(3), 判断2(5) |
| 4 | -1| -1| 1 | 0 | 判断1(4), 判断2(5) |
| 5 | 6 | 1 | -1| 7 | 判断1(1), 判断2(7:C3=F,C4=T) |
| 6 | 1 | 1 | -1| 2 | 判断1(1), 判断2(8:C3=F,C4=F) |

**最少用例**：**6 组**（因为组合1在判断1中出现了多次，但判断2需要不同组合，无法进一步合并）

> **答题技巧**：条件组合覆盖最容易丢分！一定要**逐个判断拆解条件**，列出真值表，再反推输入值。注意 `x` 是中间变量，要根据前面的分支计算出来。

## 题目 4：循环覆盖

### 题目
以下代码计算 1 到 n 的累加和，请设计满足**循环覆盖**的测试用例。

```python
def sum_n(n):
    s = 0
    i = 1
    while i <= n:
        s += i
        i += 1
    return s
```

---

### 解题过程

**循环覆盖测试要点**：
- **0 次循环**：跳过循环体
- **1 次循环**：执行一次
- **2 次循环**：执行两次（验证循环迭代）
- **m 次循环**：正常次数（如 n=5）
- **最大次数**：边界值（如 n=系统允许的最大值，这里假设为 10000）
- **最大次数+1**：n=10001（越界测试）

| 用例编号 | 输入 n | 预期结果 | 覆盖类型 |
|---------|--------|---------|---------|
| 1 | 0 | 0 | 0次循环（边界） |
| 2 | 1 | 1 | 1次循环 |
| 3 | 2 | 3 | 2次循环 |
| 4 | 5 | 15 | m次循环（正常） |
| 5 | 10000 | 50005000 | 最大次数（假设上限10000） |
| 6 | 10001 | 非法/报错 | 最大次数+1（越界） |

> **答题技巧**：循环覆盖的**标准答法**就是列这6种情况。如果题目给了具体上限，就用具体值；没给就写"假设最大为N，则测试N和N+1"。

## 总结：各方法答题模板

| 方法 | 答题时必须包含 |
|------|--------------|
| **等价类** | 有效等价类表 + 无效等价类表 |
| **边界值** | 上点/离点/内点分析 + 用例表 |
| **判定表** | 条件桩/动作桩 → 全组合表 → 合并简化 → 用例 |
| **因果图** | 因-果关系 → 因果图 → 判定表 → 用例 |
| **语句覆盖** | 控制流图 + 路径 + 用例 |
| **分支覆盖** | 每个判断的真/假分支都要走到 |
| **条件组合** | 每个条件的真值表 → 反推输入 → 用例 |
| **循环覆盖** | 0次、1次、2次、m次、最大、最大+1 |
