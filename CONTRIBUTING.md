# 致每一位潜在开源贡献者
## 很高兴like-minded的同学加入，愿开源精神在SCUT发扬。

## 项目开发阶段概览
##### 一期仓库初期规模5.73G，最终在12.9G时发觉现有方式难以满足需求，进行迭代。此期内容为1-4学期，此阶段加密采用统一字符，并未开源上机考试，并未涵盖高度复习笔记
##### ⚠目前二期仓库规模约12G，删减了大量第一学期无用冗余资源及其他已开源仓库重复代码，增加第五学期课程资料，合并通选课目录避免视图冗杂，并采用hash（sha256）分别加密，进一步缩减加密范围内容。
##### 预计第三期仓库整个规模大致会在15G左右，阶段性要求如下：
- 🟢 最低目标：将每一门课拓展出一个readme.md文件，包括三部分（可置空）：
    - 课程考察制度
    - 相关老师评价
    - 课程备考建议
- 🟡 中等目标：完善通选课部分，将包打听的锐评xlsx进行情感分析（NLP）、不稳定性排除，经过数据处理加权给分情况、考勤情况、热度等，结合各年级的学科树，做一个SCUT通选课智能推荐系统，单列于SCUT_CS，作为第三期仓库子系统。例如做一个纯前端页面，通过加载处理好的 data.json 实现筛选和搜索，部署在 GitHub Pages。**（如果数据不足或难以收集，将采用问卷形式，做一个CS选修课智能推荐系统）**
- 🔴 高等目标：**建立有完备的支付逻辑的全栈平台**部署至华为云Server。此仓库中文件打包zip分学科发布在release中，全栈项目将其作为免费OSS，通过抓github的proxy国内镜像包来实现大文件下载。其余数据储存在此项目后端，我认为flask可能就足以支撑。**如果有相关自愿捐赠，将全额用于智能基座社团资金部分流转。**（可能用oAuth,并且现有仓库可能加密系统会进行一次更迭，在字符串拼接基础上再加上一段动态盐值，或者搞一个HMAC防止彩虹表攻击）
